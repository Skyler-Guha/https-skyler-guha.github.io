Schema Based Json Validator

  • This is a demo program that demonstrates validating the datastructure and contents of a .json file using a schema file.
  • It also demonstrates creating a schema for a given problem situation.
  • It can be used to validate any .json file and can be importd as a module as well.
  • It follows the PEP8 coding standard and contains docstrings.
  • It was written based on a problem statement given to me by Third Origin LLC as part of their selection process for their internship.
  • Click here to view repository

Given Problem statement:

Validate a Json against a given schema file.

Create a class called JsonValidator. The class should be able to validate using a schema with the following properties:
  • required fields. Example id, name field to be declared as they are mandatory
  • atleast one of many fields to be present. Example one of home phone or cell phone or work phone fields
  • either one field or another field. For example: Either birth date or govt id number
  • mutually exclusive fields (if one is present, the other should not be present)
  • field value to be one of a set of values. Example field day can have only one of SU,MO,TU,WE,TH,FR,SA (enum)
Define the following method:
 validate_schema(json_file, schema_file, schema_integrity_file)
(method should return True or False if validation succeeds or fails)
